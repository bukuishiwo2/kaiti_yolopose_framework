2. 用该 checkpoint 回到 `UR Fall` 单独 fine-tune

正式评估结果：

- Recall: `0.7401`

- F1: `0.7800 -> 0.7979`

- ADL stable FP segments: `11 -> 9`

结论：
- `FallVision` 路线是可行的
- 正确用法是：
  - 受控抽样
  - 混合训练
  - 回到 `UR Fall` 做收口 fine-tune

## 当前结论

截至今天，`FallVision` 不再只是“可能有用的辅助数据源”，而是已经被验证为：
- 可用于 hard negative 增强
- 能在两阶段训练方案下提升 `UR Fall` 正式结果

当前最佳默认学习型主线为：
1. `FallVision no-fall` 视频级受控抽样
2. 与 `UR Fall` 混合训练
3. 回到 `UR Fall` 单独 fine-tune
4. 用 `UR Fall` 做正式评估

系统主线方面，当前已经具备：
1. 默认感知事件源
2. supervisor 桥接
3. planner request 边界
4. 最小任务层占位反馈
5. 电脑摄像头 ROS2 图像流接入
