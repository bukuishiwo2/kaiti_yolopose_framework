### 4. 最小接入顺序收口

当前建议顺序：

1. 复查 Phase 4a 基线：`/rgbd_odometry` 不启动、`/odom` 单 publisher、`/map` 非空、TF 连通。
2. 使用简单场景或 `warehouse` 中小范围区域作为第一轮 Nav2 baseline。
3. 只用手动 `NavigateToPose` goal，不从 `/task_planner/request` 自动派发 goal。
4. 补最小 Nav2 参数候选：frames、footprint、scan topic、planner/controller plugin、速度和加速度限制。
5. 先验收 lifecycle、costmap、`/plan`、`/cmd_vel`。
6. 通过短距离单 goal smoke test 后，再讨论正式 Nav2 launch/config 与后续 planner 接入。

### 5. Phase 4b 最小 Nav2 precheck 实现

已新增：

- `ros2_ws/src/yolopose_ros/launch/phase4b_nav2_precheck.launch.py`
- `ros2_ws/src/yolopose_ros/config/phase4b_nav2_precheck.yaml`

当前 launch 行为：

- 默认先启动 Phase 4a TurtleBot4 + RTAB-Map + perception 基线
- 强制保持 TurtleBot4 simulator 自带 `nav2=false`
- 保持 RTAB-Map `visual_odometry=false`、`publish_tf_odom=false`
- 使用 `nav2_bringup` 的 `navigation_launch.py` 启动 Nav2 navigation servers
- 不启动 `localization_launch.py`
- 不启动 `map_server / AMCL`
- 不接 `PlanSys2 / LTL`
- 不从 `/task_planner/request` 自动派发 goal

当前参数文件口径：

- `global_frame=map`
- `robot_base_frame=base_link`
- `odom_topic=/odom`
- local/global costmap 均使用 `/scan`
- global costmap 订阅 `/map`
- `map_subscribe_transient_local=false`，用于兼容 RTAB-Map 在线地图
- 速度和加速度限制采用保守值，仅用于短距离 smoke test

- Phase 4b 完成的是“基于 Phase 4a 空间层的 Nav2 最小 precheck”。
- 已证明 Nav2 lifecycle/action/costmap/plan/cmd_vel 最小链路可工作。
- 已证明 RViz2 手动初始位姿估计后，手动短距离 goal 可以到达。
- global/local costmap lifecycle 均已确认 `active [3]`；RViz2 local costmap Warn 不阻塞 Phase 4b，可留到后续正式导航稳定化时复查。

### 下一阶段建议

当前阶段不强制继续做 Phase 4c。若后续需要进入正式导航实验或任务层对接，再定义 Phase 4c：Nav2 baseline 稳定化。

可选事项：

3. 对比 RTAB-Map 在线 `/map` 与静态 map baseline。
4. 复查 RViz2 local costmap Warn 是否来自显示配置、瞬时 topic 状态或 costmap layer 状态。

## 未解决风险

- RTAB-Map 地图质量、长时间定位稳定性和 `map -> odom` 漂移尚未量化。
- 在线地图持续变化可能导致 Nav2 costmap 或 global plan 不稳定。
- 同时运行 Gazebo、RTAB-Map、YOLOPose 和 Nav2 可能带来实时性压力。
- TurtleBot4 simulator 自带 Nav2 与本项目未来 Nav2 bringup 不能同时启动。
- 真实 `PlanSys2 / LTL`、动态家居、多点任务和摔倒救援闭环仍不在当前阶段范围内。

启动 Phase 4b precheck：

```bash
cd ros2_ws
source /opt/ros/humble/setup.bash
colcon build --symlink-install --packages-select yolopose_ros
source install/setup.bash
ros2 launch yolopose_ros phase4b_nav2_precheck.launch.py
```

另开终端检查：

```bash
ros2 node list | rg 'rgbd_odometry|rtabmap|pose_stream|system_supervisor|task_planner'
ros2 topic info -v /odom
ros2 topic echo --once /map
ros2 topic echo --once /localization_pose
ros2 topic echo --once /scan
ros2 run tf2_ros tf2_echo map odom
ros2 run tf2_ros tf2_echo odom base_link
ros2 run tf2_ros tf2_echo base_link oakd_rgb_camera_optical_frame
```

检查 Nav2 precheck 输出：

```bash
ros2 lifecycle nodes
ros2 action info /navigate_to_pose
ros2 topic echo --once /global_costmap/costmap
ros2 topic echo --once /local_costmap/costmap
ros2 topic echo --once /plan
ros2 topic echo /cmd_vel
```

手动发送 1 个短距离固定 goal：

```bash
ros2 action send_goal /navigate_to_pose nav2_msgs/action/NavigateToPose \
  "{pose: {header: {frame_id: map}, pose: {position: {x: 1.0, y: 0.0, z: 0.0}, orientation: {w: 1.0}}}}"
```
