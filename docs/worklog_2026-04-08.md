### 3. 初版摔倒检测功能实现

- 在姿态识别结果基础上新增了规则法摔倒检测模块。
- 当前规则法位于：
  - `src/yolopose/pipeline/fall_detector.py`
- 使用的核心判定信号包括：
  - 人体框是否横向展开
  - 躯干是否接近平躺
  - 下肢是否接近平躺或收折
- 判定逻辑为：
  - 三个信号中至少满足两个，判为当前帧疑似跌倒
- 为减少抖动，又加入了时序稳定器：
  - 连续若干帧为真才触发稳定跌倒
  - 连续若干帧为假才取消跌倒状态

## 方法说明

### 姿态识别方法

- 当前没有重新训练人体姿态模型。
- 直接采用官方预训练 `YOLOPose` 模型作为前端特征提取器。
- 模型负责输出人体框和 17 个关键点。

### 摔倒检测方法

- 当前没有训练专门的跌倒分类网络。
- 采用的是“基于关键点几何关系的规则法”。
- 主要依赖人体姿态在跌倒场景下常见的几何特征。
- 再通过时序稳定器将单帧判断转换为事件级输出。

### 评估方法

- 使用 `UR Fall Detection Dataset` 进行验证。
- 同时统计：
  - 帧级 Precision / Recall / F1
  - 段级误报与漏报
  - 误报次数
  - 检测延迟
- 通过 `fall` 样本和 `adl` 样本分别观察召回率和误报率。

### 2.1 `predict` 调参结果

以当前基线 `c010` 为参考：

- 基线 `c010`

  - Recall: `0.4196`
  - F1: `0.5396`

本轮 `predict` 网格搜索中，三组结果最值得记录：

1. 综合 F1 最优：`c009`
- 参数：
  - `keypoint_conf_threshold=0.3`
  - `torso_vertical_ratio_threshold=0.22`
  - `hip_to_ankle_vertical_ratio_threshold=0.38`
  - `min_signals_to_fall=2`
  - `min_true_frames=3`
  - `min_false_frames=8`
- 指标：
  - Precision: `0.6731`
  - Recall: `0.4891`
  - F1: `0.5665`
  - `fall_fn_segments=3`
  - `adl_fp_segments=22`
- 说明：
  - 与基线相比，召回提升明显
  - 只需将稳定触发帧数从 `5` 调低到 `3`
  - 是当前规则法下最合理的折中方案

2. 排名第一：`c016`
- 参数：
  - `keypoint_conf_threshold=0.3`
  - `torso_vertical_ratio_threshold=0.26`
  - `hip_to_ankle_vertical_ratio_threshold=0.45`
  - `min_signals_to_fall=2`
  - `min_true_frames=5`
  - `min_false_frames=8`
- 指标：
  - Precision: `0.5755`
  - Recall: `0.5148`
  - F1: `0.5435`
  - `fall_fn_segments=2`
  - `adl_fp_segments=26`
- 说明：
  - 该结果按“先少漏检段、再少误报段”的排序规则排第一
  - 更适合“宁可多报，也尽量少漏”的场景

3. 召回更激进：`c015`
- 指标：

- 说明：

  - 但误报显著增加，不适合直接作为主配置

## 目前存在的问题

1. 当前摔倒检测仍然依赖人工规则，泛化能力有限。
2. 对复杂动作、躺下、弯腰、翻滚等边界动作区分能力不足。
3. 当前召回率仍偏低，说明漏检问题还比较明显。
4. `track` 方案暂时没有优于 `predict`。
5. 还没有训练专门的时序跌倒识别模型。
