# Method Roadmap

本文件用于回答一个核心问题：在当前 `YOLOPose + LSTM` 已完成初步验证后，后续最合适的方法路线是什么。

## 1. 当前阶段判断

当前项目已经完成：
- 规则法 baseline
- 学习型时序模型（LSTM）
- 两轮阈值与稳定参数调参
- 与规则法的同口径对照

当前结论：
- `LSTM` 已明显优于规则法
- 当前主要问题是 ADL 误报
- 误报动作集中在：
  - 躺下
  - 坐下
  - 弯腰
  - 地面动作

这意味着项目已经不再处于“先把链路跑通”的阶段，而进入“如何提高动作区分能力”的阶段。

## 2. 方法选择原则

当前最重要的不是立刻追求更复杂的网络，而是控制变量。

方法优先级应遵循：
1. 先确认当前数据与标签是否足以支撑更复杂模型
2. 先把 hard negative 问题讲清楚
3. 再在相同数据上逐步升级时序模型

如果跳过这一步，直接上更复杂架构，常见结果是：
- 训练更慢
- 调参成本更高
- 误报问题仍然存在
- 很难解释性能变化究竟来自模型还是数据

## 3. 当前最合适的候选路线

### 3.1 路线 A：继续用当前关键点序列框架，但补强负样本

这是当前最推荐的路线。

做法：
- 补充或构造更多 hard negative
- 重点覆盖：
  - 躺下
  - 坐下
  - 弯腰
  - 捡东西
  - 地面活动
- 保持现有 `YOLOPose -> 序列特征 -> LSTM` 框架不变
- 重新训练并复评

优点：
- 成本最低
- 变量最少
- 最适合确认“问题究竟是不是数据不足”

结论：
- 应该作为下一阶段的第一优先级

### 3.2 路线 B：在同一套数据上替换为 `TCN`

这是当前最值得尝试的模型升级方向。

原因：
- `TCN` 对固定长度序列通常很稳
- 比 `LSTM` 更容易并行
- 参数规模可控
- 工程复杂度明显低于 `Transformer` 和 `ST-GCN`

适用场景：
- 你已经确认 hard negative 会长期存在
- 想在不引入过多复杂度的前提下做下一步模型升级

结论：
- 如果要升级模型，优先 `TCN`

### 3.3 路线 C：Transformer 类模型

Transformer 方向不是不能做，但当前不建议立刻作为第一升级路线。

原因：
1. 当前数据规模不大
- Transformer 通常更吃数据量
- 小数据集下更容易过拟合

2. 当前问题是“跌倒 vs 躺下/坐下/弯腰”的细粒度区分
- 这类问题未必需要更大的模型
- 很多时候先补 hard negative 比换 Transformer 更有效

3. 工程成本更高
- 训练和调参更复杂
- 很容易陷入“模型更复杂但收益不稳定”

什么时候适合上 Transformer：
- 你后续拿到了更多视频和更丰富的负样本
- `LSTM / TCN` 已经明显到瓶颈
- 你希望探索更长时序依赖和更强表达能力

结论：
- Transformer 可以作为中后期研究方向
- 但现在不应作为第一优先级

### 3.4 路线 D：ST-GCN / Skeleton GCN

这是骨架序列任务里很自然的方向，但目前也不建议直接作为第一步。

优点：
- 更贴合人体关键点图结构
- 对骨架动作识别任务理论上更自然

缺点：
- 工程复杂度比 `LSTM / TCN` 高
- 训练和数据组织更复杂
- 如果当前问题主要来自数据负样本不足，收益可能不如预期

结论：
- 它是合理的长期研究方向
- 但优先级仍低于“补 hard negative + 先试 TCN”

## 4. 当前推荐的研究顺序

建议按以下顺序推进：

1. 固化当前 `LSTM` 结果
- 将 `score_threshold=0.6` 的配置作为正式 baseline

2. 补 hard negative
- 收集或整理躺下、坐下、弯腰、地面动作视频
- 重新训练当前 LSTM

3. 在相同数据条件下试 `TCN`
- 保持相同输入特征
- 和当前 LSTM 做同口径比较

4. 如收益仍不够，再考虑：
- Transformer
- ST-GCN

## 5. 当前建议

如果目标是“稳妥推进研究，而不是跳过太多步骤”，当前最合理的选择是：

1. 不再继续深挖当前 LSTM 阈值微调
2. 先记录当前 LSTM 已达到的最好结果
3. 下一步优先做：
- hard negative 数据补强
- `TCN` 实现

Transformer 方向值得做，但更适合作为后续升级路线，而不是现在立刻切换的第一目标。
