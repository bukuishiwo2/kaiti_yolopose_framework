# Dataset Positioning And External Validation

本文件用于统一当前项目的数据集定位，避免“主 benchmark”“补充训练集”“外部验证集”在不同文档中混写。

## 1. 当前默认数据策略

当前默认感知主线不变：

- 主学习模型：`LSTM`
- 默认模型：`models/fall_sequence_lstm_urfall_finetune_from_fallvision_sampled.pt`
- 当前正式结果：`Precision=0.8657`、`Recall=0.7401`、`F1=0.7979`

在数据层，当前统一定位如下：

### 1.1 `UR Fall`

定位：

- 主 benchmark
- 主训练与主评估收口集
- 当前正式结果的基准数据集

原因：

- 当前训练、评估、benchmark 与系统联调结论都围绕它建立
- 仓库中的默认结论必须继续可复现、可追溯

### 1.2 `FallVision no-fall`

定位：

- hard negative 补充源
- 受控抽样辅助数据

当前正式口径：

- 不适合全量直接并入训练
- 当前可接受方案是视频级受控抽样，并在 `UR Fall` 上回灌 fine-tune

### 1.3 规则法 baseline

定位：

- baseline
- debug 与对照

规则法不是数据集，但需要强调：

- 它不再是系统层默认主决策
- 仍保留为评估与联调的重要对照分支

## 2. 当前最需要补的不是“换主数据集”

从当前线上与离线现象看，主要短板是：

- 慢跌倒
- 家居场景跌倒
- 家居 ADL 误报
- 外部视频域泛化

因此当前更合理的策略是：

- 保留 `UR Fall` 作为主 benchmark
- 用额外数据补盲区样本
- 不直接推翻现有主闭环

## 3. 补充训练集定位

### 3.1 `FallVision fall`

建议定位：

- 第一优先补充训练集

建议优先覆盖：

- `chair fall`
- `bed fall`
- `standing slow fall`

原因：

- 这些样本更接近家居场景中的慢跌倒与控制性倒地
- 与当前项目目标比，价值高于继续只增加 no-fall

### 3.2 训练使用原则

建议顺序：

1. 保持 `UR Fall` 主线不动
2. 小规模加入 `FallVision fall + no-fall`
3. 做受控 fine-tune
4. 回到 `UR Fall` 做正式主评估

不建议当前直接做的事：

- 用新数据集完全替换 `UR Fall`
- 把新增数据全量一锅炖并入训练
- 因外部视频个例直接推翻当前默认主线

## 4. External Validation 定位

`external validation` 指：

- 不一定进入主训练
- 但用于验证域外泛化能力的数据集或视频源

当前优先关注的外部验证目标：

- 慢跌倒
- 家居风格视频
- 多视角
- 遮挡
- 半自然或非受控场景

建议定位：

### 4.1 `Le2i`

- 优先作为视频域外部验证集
- 用于检查多环境、多视角下的泛化能力

### 4.2 家居 / 非受控 / 半自然 fall 数据

- 优先作为家居域与遮挡域验证集
- 在没有完成验证前，不建议直接并入主训练

## 5. 与系统主线的关系

数据策略必须服务于系统目标，而不是脱离系统单独扩张。

当前系统主线真正需要的是：

- 可被 supervisor 稳定消费的默认跌倒语义
- 对家居场景与慢跌倒更稳的主线
- 对遮挡和域外视频更透明的外部验证

因此，数据策略当前应回答三类问题：

1. 主 benchmark 上是否仍稳定
2. 慢跌倒 recall 是否提高
3. 外部家居/遮挡场景泛化是否改善

## 6. 当前阶段建议

当前建议按以下优先级推进：

1. 保留 `UR Fall` 作为主 benchmark 与主收口集
2. 优先补 `FallVision fall` 正样本
3. 保留 `FallVision no-fall` 作为 hard negative
4. 将 `Le2i` 与家居/非受控数据先作为外部验证
5. 在外部验证稳定后，再决定是否小规模进入训练
